
================================================================
  SCENARIO 2 — one remote, two clones (file:// transport)
================================================================

---- Create the 'remote' — a dolt remote is actually a directory of chunks ----
(the remote is just /home/developer/gits/dolt-explainer/runs/02-file-remote/remote-store — no server, just a filesystem path)

---- Clone A: initialize, seed, add the file:// remote ----
$ dolt remote add origin file:///home/developer/gits/dolt-explainer/runs/02-file-remote/remote-store
$ dolt remote -v
origin file:///home/developer/gits/dolt-explainer/runs/02-file-remote/remote-store 

---- Push from Clone A — this materializes chunk files under /home/developer/gits/dolt-explainer/runs/02-file-remote/remote-store ----
$ dolt push -u origin main
