================================================================
  CONCLUSION
================================================================
1. dolt remote add + fetch + push with multiple remotes works like git.
2. A GitHub PR from fork -> upstream shows an EMPTY diff, because the
   web UI only looks at refs/heads/* which are typically identical.
3. The GitHub 'Merge PR' button only fast-forwards refs/heads/main.
   refs/dolt/data on upstream stays stale — the data change is LOST
   from upstream's perspective unless someone with upstream-write pulls
   locally and pushes refs/dolt/data up.
4. For Igor's case (ai-tools has fork-write only), the admin (idvorkin)
   must pull from fork + dolt merge + dolt push upstream. No pure-
   GitHub workflow exists on vanilla refs. DoltHub solves this by
   making PRs dolt-aware.
